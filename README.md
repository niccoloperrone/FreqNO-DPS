# FreqNO-DPS

**Correcting Neural Operator Spectral Bias via Diffusion Posterior Sampling with Sparse Observations**

[![arXiv](https://img.shields.io/badge/arXiv-2606.03936-b31b1b.svg)](https://arxiv.org/abs/2606.03936)
[![License: GPL v3](https://img.shields.io/badge/License-GPL%20v3-blue.svg)](LICENSE)

Niccolò Perrone, Fanny Lehmann, Stefania Fresca, Filippo Gatti

Reference implementation of **FreqNO-DPS**, a diffusion posterior sampling
framework that corrects the spectral bias of neural operator surrogates
by combining an unconditional score-based diffusion prior with sparse
sensor observations and a frozen neural operator prediction. The neural
operator enters the posterior as a parallel observation channel with a
frequency-calibrated likelihood: its information is admitted at the
bands where it is reliable, and the sensor channel takes over at the
bands where the surrogate's bias is most severe.

We demonstrate the framework on 3D elastic surface wavefield
reconstruction from sparse sensor coverage (5% and 2%), where it
achieves near-zero spectral bias across all frequency bands.

![Overview of FreqNO-DPS](https://arxiv.org/html/2606.03936v1/x1.png)

## 1. Setup

Clone the repository and create a virtual environment:

```bash
git clone https://github.com/niccoloperrone/FreqNO-DPS.git
cd FreqNO-DPS

python3 -m venv freqnodps
source freqnodps/bin/activate
pip install -r requirements.txt
```

Tested with Python 3.10, PyTorch 2.4, CUDA 12.1.

The `MIFNO/` subdirectory is the vendored MIFNO codebase from
[lehmannfa/MIFNO](https://github.com/lehmannfa/MIFNO) (GPL-3.0).

## 2. Data

We use the **HEMEWS-3D** dataset from
[Lehmann et al., 2024](https://essd.copernicus.org/articles/16/3949/2024/):
3D surface particle velocity wavefields on a 32×32 grid over 320 time
steps (Δt = 0.02 s), generated by Spectral Element Method simulations
of seismic wave propagation in heterogeneous elastic media.

### 2.1 Download and preprocess

Full instructions live in [`MIFNO/data/readme.md`](MIFNO/data/readme.md).
Summary:

1. Create `data/raw/` and `data/formatted/`.
2. Download HEMEWS-3D into `data/raw/` (see the MIFNO readme for the
   data repository link).
3. From `MIFNO/data/`, run:
   ```bash
   python3 create_data_geologies.py    --Ntrain 27000 --Nval 3000 --Ntest 1000
   python3 create_data_sources.py      --Ntrain 27000 --Nval 3000 --Ntest 1000
   python3 create_data_velocityfields.py --Ntrain 27000 --Nval 3000 --Ntest 1000
   ```
4. The preprocessed `sample_i.h5` files appear under `data/formatted/`,
   along with `a_mean.npy` and `a_std.npy`.

### 2.2 Z-score statistics

The dataset class normalizes targets channel-wise using mean and std
computed over the training split (see `dataloader/dataset.py`,
constant `VEL_STATS_PATH`). You have two options:

**Option A — use ours (recommended).** Download
`vel_zscore_stats_train.npz` from the pretrained checkpoints bundle
(see Section 3) and place it at
`./checkpoints/vel_zscore_stats_train.npz`. No code changes needed.

**Option B — compute from scratch.** After preprocessing, run:

```bash
python compute_zscore_stats.py \
    --dirs ./data/formatted/HEMEWS3D_S32_Z32_T320_fmax5_rot0_train \
    --out ./checkpoints/vel_zscore_stats_train.npz
```

### 2.3 Data path

The training dataloader looks under
`./data/HEMEWS3D_S32_Z32_T320_fmax5_rot0_train` by default
(constant `DEFAULT_TRAIN_DIR` in `utils/gencfd_utils.py`). Either move
the preprocessed folder there, or edit the constant to point at
`./data/formatted/`. Same applies to the validation/test splits used by
the eval scripts — paths are at the top of each script.

## 3. Pretrained checkpoints

Pretrained checkpoints and statistics are hosted on **Hugging Face**
[**[link to be added]**](#). Download and place under `./checkpoints/`:

```
./checkpoints/
├── checkpoint_430000.pth        # unconditional GenCFD diffusion prior
├── mifno_3d_S32_T320.pt         # frozen MIFNO surrogate
├── vel_zscore_stats_train.npz   # per-channel z-score statistics
├── spectral_model.pt            # H(c,k), sigma2_NO(c,k), P_u(c,k)
└── sigma2_iso.pt                # isotropic sigma2_NO for the DPS_NO_iso baseline
```

## 4. Training

We provide the pretrained checkpoint, but to reproduce training from
scratch on 8× A100 80GB:

```bash
torchrun --nproc_per_node=8 train/train_prior.py \
    --batch_size 4 \
    --num_train_steps 430000 \
    --save_dir ./runs/gencfd_unconditional
```

This trains the unconditional GenCFD denoiser (430k steps, batch size
32 total) under the variance-exploding EDM scheme with σ_min = 0.002,
σ_max = 80, log-uniform noise sampling, AdamW with peak LR 3×10⁻⁴,
weight decay 0.01, and EMA decay 0.999.

Checkpoints are saved under `./runs/gencfd_unconditional/checkpoints/`.

## 5. Inference

Inference is a two-step pipeline: **(5.1)** posterior sample generation
on GPU, then **(5.2)** CPU-only metrics aggregation. Splitting them lets
you re-run metrics without redoing the expensive sampling step.

The supported methods and samplers are:

| Method            | Surrogate guidance              | λ_sensor (ODE)  | λ_NO (ODE) |
|-------------------|---------------------------------|-----------------|------------|
| `FreqNO_DPS`      | Spectrally shaped (**ours**)    | 23 000          | 0.35       |
| `DPS_sensor_only` | None — sparse sensors only      | 23 000          | —          |
| `DPS_NO_iso`      | Isotropic L2 (DPS baseline)     | 23 000          | 10 000     |

SDE hyperparameters are halved relative to ODE values.

### 5.1 Generate posterior samples

Edit the constants at the top of `eval/generate.py`:

```python
METHOD    = "FreqNO_DPS"       # or "DPS_sensor_only", "DPS_NO_iso"
WHAT_DPS  = "ODE"              # or "SDE"
KEEP_FRAC = 0.05               # 0.05 for 5% coverage, 0.02 for 2%
```

Then run on a single GPU:

```bash
python eval/generate.py --model_dir ./checkpoints
```

Or on multiple GPUs:

```bash
torchrun --nproc_per_node=4 eval/generate.py --model_dir ./checkpoints
```

Per-rank partials are written under
`./results/<run>/partials/part_rank<r>.pt`. The full test set takes
about an hour and half on 4× A100.

### 5.2 Compute metrics

After generation completes, aggregate the partials and compute metrics
on a single CPU process:

```bash
python eval/compute_metrics.py \
    --partials_dir ./results/<run>/partials
```

A CSV with rMAE, rRMSE, rFFT in three frequency bands (0–1 Hz,
1–2 Hz, 2–5 Hz), and significant duration (D5–95) is written to
`./results/<run>/`.

This step is cheap (minutes) and idempotent, so you can re-run it
freely after editing `eval/metrics.py` or adding new diagnostics
without redoing the GPU step.

### 5.3 Posterior calibration

To measure posterior coverage and concentration:

```bash
python eval/eval_posterior_calibration.py --model_dir ./checkpoints
```

Runs K = 20 posterior realizations per sample over N = 100 samples and
reports 1σ and 2σ coverage, mean confidence interval width, and
per-realization rMAE statistics.

## 6. Spectral calibration and diagnostics

The spectral model (transfer function, residual variance, signal power
spectrum) is precomputed in the checkpoint bundle. To recompute on a
different MIFNO version or different calibration data:

```bash
python spectral_model/estimate_spectral_model.py \
    --h5 ./data/MIFNO_outputs_calibration.h5 \
    --stats ./checkpoints/vel_zscore_stats_train.npz \
    --out ./checkpoints/spectral_model.pt

python spectral_model/estimate_sigma2_iso.py \
    --h5 ./data/MIFNO_outputs_calibration.h5 \
    --stats ./checkpoints/vel_zscore_stats_train.npz \
    --out ./checkpoints/sigma2_iso.pt
```

The calibration split must be disjoint from training and test.

The spectral observation model approximates the NO residual covariance
as diagonal in the Fourier domain (wide-sense stationarity). To verify
this on your data:

```bash
python wss/verify_wss.py \
    --h5 ./data/MIFNO_outputs_val.h5 \
    --model ./checkpoints/spectral_model.pt \
    --stats ./checkpoints/vel_zscore_stats_train.npz \
    --out ./results/wss_verification
```

## 7. Paper figures

Spectral calibration figures (relative error profile, transfer function,
LMMSE vs DPS weights, expected guidance):

```bash
python eval/plot_spectral_calibration.py \
    --model ./checkpoints/spectral_model.pt \
    --sigma2_iso ./checkpoints/sigma2_iso.pt
```

WSS coherence figure (after Section 6):

```bash
python wss/plot_wss.py \
    --input ./results/wss_verification/wss_coherence_raw.npz \
    --output ./figures/fig_wss_coherence.pdf
```

## Repository structure

```
FreqNO-DPS/
├── train/                       unconditional diffusion prior training
│   ├── train_prior.py           entry point
│   ├── trainers.py
│   ├── training_loop.py
│   └── train_states.py
├── eval/                        posterior sampling and evaluation
│   ├── generate.py              GPU sampling, writes per-rank partials
│   ├── compute_metrics.py       CPU aggregation, writes metrics CSV
│   ├── eval_posterior_calibration.py
│   ├── plot_spectral_calibration.py
│   └── metrics.py, mifno_helpers.py, normalization.py, sensor_utils.py
├── spectral_model/              transfer function and residual-variance estimators
├── wss/                         wide-sense stationarity verification
├── dps/                         likelihoods and measurement operators (ours)
├── diffusion/                   diffusion scheme, samplers, guidance
├── solvers/                     ODE and SDE solvers
├── model/                       GenCFD U-Net architecture (Apache-2.0)
├── dataloader/                  HEMEWS-3D dataset reader
├── utils/                       shared helpers (DDP, parser, callbacks, ...)
├── MIFNO/                       vendored MIFNO (GPL-3.0)
├── compute_zscore_stats.py      one-time z-score statistic computer
├── LICENSE                      GPL-3.0
├── NOTICE                       third-party attributions
└── requirements.txt
```

The DPS samplers (`DPSOdeSampler`, `DPSSampler` in
`diffusion/samplers.py`) and the likelihoods that go into them
(`dps/likelihoods.py`) are original work and entirely separable from
the GenCFD scaffolding.
